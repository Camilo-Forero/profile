Different technologies for different solutions

In my time as a full developer i learn to use different technologies to get the best result and aproach to the desired solution always try to think out the box sometimes that's the best way to go.

Exist times were the optimal and manteinable solution is the basic and easy one and always rember this:

  • The purpose of software engineering is to reduce complexity, not to create it. "Keep It Simple, Stupid (KISS)"
  • Try to not duplicate the code and make it maintenable. “Don’t Repeat Yourself (DRY)”
  • Dont write code that you think you gonna need in the future “You Aren’t Gonna Need It (YAGNI)”
  • Try to partition a complicate application in different parts “Separetion if Concerns (SoC)”
  • Avoid premature optimizations
